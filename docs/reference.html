API Reference#

class AsyncBox#
async def async_fn(b1, b2):
    args, kwargs = await b1.get()
    assert args == (1, )
    assert kwargs == {'crow': 'raven', }

    args, kwargs = await b2.get()
    assert args == (2, )
    assert kwargs == {'frog': 'toad', }

    args, kwargs = await b1.get()
    assert args == (1, )
    assert kwargs == {'crow': 'raven', }

b1 = AsyncBox()
b2 = AsyncBox()
b1.put(1, crow='raven')
start(async_fn(b1, b2))
b2.put(2, frog='toad')

Warning

This class is not designed for inter-task synchronization, unlike asyncio.Event. When multiple tasks simultaneously try to get an item from the same box, it will raise an exception.

clear()#

Remove the item from the box if there is one.

get()#

Get the item from the box if there is one. Otherwise, wait until it’s put.

property is_empty: bool#

Whether the box is empty.

put(*args, **kwargs)#

Put an item into the box if it’s empty.

put_or_update(*args, **kwargs)#
update(*args, **kwargs)#

Replace the item in the box if there is one already.

class AsyncEvent#
async def async_fn(e):
    args, kwargs = await e.wait()
    assert args == (2, )
    assert kwargs == {'crow': 'raven', }

    args, kwargs = await e.wait()
    assert args == (3, )
    assert kwargs == {'toad': 'frog', }

e = AsyncEvent()
e.fire(1, crocodile='alligator')
start(async_fn(e))
e.fire(2, crow='raven')
e.fire(3, toad='frog')

Warning

This class is not designed for inter-task synchronization, unlike asyncio.Event. When multiple tasks simultaneously try to wait for the same event to fire, it will raise an exception.

class CancelScope#

An equivalence of trio.CancelScope. You should not directly instantiate this, use open_cancel_scope().

cancel()#

Cancel the execution inside this scope as soon as possible.

cancel_called#

Whether the cancel() has been called.

cancelled_caught#

Whether the scope caught a corresponding Cancelled instance.

property closed: bool#

Whether this scope has been closed. The cause of the closure of the scope can be either an exception occurred or the scope exited gracefully,

Cancelled#

Exception class that represents cancellation. See Dealing with Cancellation.

Warning

Actually, this is not an exception class but a tuple of exception classes for now. But that’s an implementation detail, and it might become an actual class in the future; therefore, your code must be compatible in both cases.

class Event#

Similar to asyncio.Event. The differences are:

  • set() accepts any number of arguments but doesn’t use them at all so it can be used as a callback function in any library.

  • is_set is a property not a function.

e = Event()
any_library.register_callback(e.set)

Deprecated since version 0.6.2: This class is deprecated, and will be removed before 1.0.0. Use asyncgui_ext.synctools.event.Event instead.

clear()#

Unset the event.

property is_set: bool#
set(*args, **kwargs)#

Set the event. Unlike asyncio’s, all tasks waiting for this event to be set will be resumed immediately.

wait() → Awaitable#

Wait for the event to be set. Return immediately if it’s already set.

exception InvalidStateError#

The operation is not allowed in the current state.

class Nursery#

Similar to trio.Nursery. You should not directly instantiate this, use open_nursery().

close()#

Cancel all the child tasks in the nursery as soon as possible.

property closed: bool#
start(aw: Awaitable | Task, /, *, daemon=False) → Task#

Immediately start a Task/Awaitable under the supervision of the nursery.

If the argument is a Task, itself will be returned. If it’s an typing.Awaitable, it will be wrapped in a Task, and that Task will be returned.

The daemon parameter acts like the one in the threading module. When only daemon tasks are left, they get cancelled, and the nursery closes.

class Task(aw: Awaitable, /)#
cancel()#

Cancel the task as soon as possible.

property cancelled: bool#

Whether the task has been cancelled.

close()#

An alias for cancel().

property finished: bool#

Whether the task has been completed.

property result: Any#

Result of the task. If the task is not finished, InvalidStateError will be raised.

property root_coro: Coroutine#

The starting point of the coroutine chain for the task. This exists solely for inspection purposes.

property state: TaskState#

The current state of the task. This exists solely for inspection purposes.

property uid: int#

An unique integer assigned to the task. This exists solely for inspection purposes.

class TaskState#

Enum class that represents the Task state.

CANCELLED#

The execution has been cancelled. The cause of cancellation can be either Task.cancel() or an unhandled exception.

CREATED#

Waiting to start execution.

FINISHED#

The execution has been completed.

STARTED#

Currently running or suspended.

current_task() → Awaitable[Task]#

Returns the Task instance corresponding to the caller.

task = await current_task()
class disable_cancellation#

Return an async context manager that protects its code-block from cancellation.

async with disable_cancellation():
    await something  # <- never gets cancelled
dummy_task = <asyncgui.Task object>#

An already closed task. This can be utilized to prevent the need for the common null validation mentioned below.

Before:

class MyClass:
    def __init__(self):
        self._task = None

    def restart(self):
        if self._task is not None:
            self._task.cancel()
        self._task = asyncgui.start(self.main())

    async def main(self):
        ...

After:

class MyClass:
    def __init__(self):
        self._task = asyncgui.dummy_task

    def restart(self):
        self._task.cancel()
        self._task = asyncgui.start(self.main())

    async def main(self):
        ...

start(aw: Awaitable | Task, /) → Task#

Immediately start a Task/Awaitable.

If the argument is a Task, itself will be returned. If it’s an typing.Awaitable, it will be wrapped in a Task, and that Task will be returned.

async def async_func():
    ...

task = start(async_func())
async wait_all(*aws: Iterable[Awaitable | Task]) → Awaitable[Sequence[Task]]#

Run multiple tasks concurrently, and wait for all of them to end. When any of them raises an exception, the others will be cancelled, and the exception will be propagated to the caller, like trio.Nursery.

tasks = await wait_all(async_fn1(), async_fn2(), async_fn3())
if tasks[0].finished:
    print("The return value of async_fn1() :", tasks[0].result)
wait_all_cm(aw: Awaitable | Task) → AsyncContextManager[Task]#

The context manager form of wait_all().

async with wait_all_cm(async_fn()) as bg_task:
    ...
async wait_any(*aws: Iterable[Awaitable | Task]) → Awaitable[Sequence[Task]]#

Run multiple tasks concurrently, and wait for any of them to finish. As soon as that happens, the others will be cancelled. When any of them raises an exception, the others will be cancelled, and the exception will be propagated to the caller, like trio.Nursery.

tasks = await wait_any(async_fn1(), async_fn2(), async_fn3())
if tasks[0].finished:
    print("The return value of async_fn1() :", tasks[0].result)
wait_any_cm(aw: Awaitable | Task) → AsyncContextManager[Task]#

The context manager form of wait_any(), an equivalence of trio_util.move_on_when().

async with wait_any_cm(async_fn()) as bg_task:
    ...
